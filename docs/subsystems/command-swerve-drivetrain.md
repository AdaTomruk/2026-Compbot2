# CommandSwerveDrivetrain

## Overview

`CommandSwerveDrivetrain` is the CTRE Phoenix 6 swerve drivetrain wrapper generated by Tuner X. It extends `TunerSwerveDrivetrain` and implements `Subsystem` so it can be scheduled by WPILib commands. It is responsible for:

- Owning the swerve modules and drivetrain state
- Maintaining odometry and pose estimation
- Integrating PathPlanner AutoBuilder configuration
- Supporting CTRE SysId characterization routines

## Key responsibilities

- **Operator perspective:** Applies field-forward orientation based on alliance color when disabled (or at first boot) to keep driver controls consistent.
- **PathPlanner integration:** Configures `AutoBuilder` with a holonomic controller and CTRE feedforward hooks.
- **SysId characterization:** Exposes translation, steer, and rotation SysId routines to tune PID gains.
- **Simulation loop:** Runs a 4 ms simulation notifier to keep sim dynamics stable.

## Primary APIs

- `applyRequest(Supplier<SwerveRequest>)`: Returns a command to set CTRE swerve control requests (field-centric, robot-centric, etc.).
- `sysIdQuasistatic(Direction)` / `sysIdDynamic(Direction)`: Run SysId tests for the currently-selected routine.
- `addVisionMeasurement(...)`: Injects vision pose updates with the proper timestamp conversion.
- `samplePoseAt(timestamp)`: Query pose history for latency-compensated vision.

## Control and configuration

- **AutoBuilder:** Uses `RobotConfig.fromGUISettings()` and a `PPHolonomicDriveController` with PID constants (translation: 10, rotation: 7).
- **Alliance flipping:** PathPlanner paths are flipped when on the red alliance.
- **Perspective:** `setOperatorPerspectiveForward(...)` is applied based on DS alliance when disabled.

## Telemetry and logging

- SysId routines log state to `SignalLogger` (translation, steer, rotation).

## Simulation

- In simulation, a `Notifier` runs at 4 ms, calling `updateSimState(...)` with battery voltage.

## Safety notes

- Operator perspective is only re-applied while disabled after the first application, avoiding sudden changes while enabled.
